Data Retention and Deletion Policy

Plaid Security Questionnaire — Question 11 Supporting Documentation
Version 1.1 · Effective Date: February 27, 2026 · Last Reviewed: March 2, 2026

Summary
RAG Consulting LLC maintains a defined and enforced data retention and deletion policy that complies with applicable data privacy laws, including CCPA/CPRA. The policy covers all consumer financial data collected through the Financial Burndown Tracker application, including data received through the Plaid API. The policy is reviewed periodically and was last reviewed on March 2, 2026.

1. Purpose

This policy defines how RAG Consulting LLC ("the Company") retains, archives, and deletes consumer financial data collected through the Financial Burndown Tracker application, including data received through the Plaid API.

2. Scope

This policy applies to all consumer data stored in:

  • Amazon S3 — application state, transaction data, parsed statements
  • Amazon DynamoDB — Plaid access tokens, user records, sync metadata, consent records
  • Any temporary storage or caches used during processing

4. Data Deletion Procedures

4.1 Bank Account Disconnection

When a user disconnects a linked bank account through the application:

  1. The Plaid access token for that institution is revoked via client.itemRemove() at Plaid.
  2. The encrypted access token is deleted from DynamoDB.
  3. The sync cursor for that item is deleted from DynamoDB.
  4. No further data is fetched from that institution.

4.2 Full Account Deletion

When a user requests account deletion, the following steps are executed:

1 All linked Plaid items are revoked via Plaid's itemRemove() API.
2 All Plaid access tokens are deleted from DynamoDB.
3 All S3 objects under the user's organization prefix (orgs/{orgId}/) are permanently deleted — including data.json, statements, and any associated files.
4 All DynamoDB records for the user (tokens, cursors, metadata, consent records) are removed.
5 The user record itself is deleted from the DynamoDB users table.
6 Deletion is completed within 30 calendar days of the request.
7 A confirmation is sent to the user upon completion.

4.3 Deletion Request Methods

Users may request data deletion by:

  • In-App: Using the "Delete Account" feature on the Settings page within the application.
  • Email: Sending a request to privacy@rag-consulting.com with the subject "Data Deletion Request."

All deletion requests are logged and tracked to completion.

4.4 Consent Records

When a user provides consent (e.g., for Plaid data access, privacy policy acceptance, or account registration), a timestamped consent record is stored on their user profile. These records include the consent type, policy version, timestamp, and user-agent. Consent records are retained for the duration of the user's active account and are permanently deleted upon account deletion.

6. Data Backup and Recovery

  • Data stored in DynamoDB benefits from AWS point-in-time recovery (PITR) when enabled.
  • Backups are retained for the minimum period required for disaster recovery.
  • When a user requests data deletion, the deletion is applied to all backup copies within the backup retention window.

7. Compliance

This policy supports compliance with:

  • Plaid's data handling requirements — consumer financial data is treated as Confidential and handled per Plaid's contractual obligations.
  • California Consumer Privacy Act (CCPA) and California Privacy Rights Act (CPRA) — consumers can request access, deletion, and portability of their data; we do not sell personal information.
  • Applicable state and federal data privacy laws.
  • The Company's Information Security Policy and published Privacy Policy.

8. Policy Review

This policy is reviewed annually and updated as necessary to reflect changes in:

  • Data processing activities
  • Regulatory requirements
  • Infrastructure changes
